## 固定プロトコル草案（v0.1）

本節は Phase A〜C の単一ノード実証の**外**で、**再現可能な最小手順**として将来の実装・論文に接続するための草案である。未実装の前提を含む。

### スコープと非スコープ

- **含む**: 複数プロセス／複数ノード間で**共鳴場の状態**（またはその圧縮表現）を**どの周期で交換するか**、**観測する指標**（遅延・ドリフト・ローカル場のエネルギー等）、**障害時のフォールバック**（1 ノード単体への縮退）。
- **含まない**: 本リポジトリの `core/` 全体の分散対応改修、商用クラスタオーケストレーション、**本番 KV 階層の完全実装**（Phase 3 の「スタブ実証」とは別軸）。

### 観測の最小セット（提案）

| 観測名 | 意味 | 記録形式の例 |
|--------|------|----------------|
| 同期バリア遅延 | ラウンド間の待ち時間 | 秒（p50/p95） |
| 場ベクトル差分 | ノード間の場表現の L2 またはコサイン距離 | スカラー／ヒストグラム |
| スループット | 単位時間あたりのトークンまたは共鳴更新回数 | tokens/s |

### ハード・トポロジ（例）

- **エッジ**: Jetson Orin 等、**同一 PyTorch ビルド**（または MPS／CUDA の明示）を前提に、**1 本のベンチ**から相対比を取る。
- **トポロジ**: 星型（中央集約）／リング（逐次同期）の**どちらを採用するか**を実験前に固定し、**変更時は改訂履歴**に残す。

### `core/` との接続方針（提案）

- **境界**: `ResonantCore` や `LightweightResonanceFacade` の**入出力テンソル形状**を変えず、**ノード間メッセージ**は「場の更新に必要な最小統計量」に限定する（全次元の生ベクトルを常時送らないオプション）。
- **単一ノード回帰**: 分散を切ったときに**単一ノードの Phase A 実証**と同じ JSON スキーマに戻せること（[module_benchmark_map.md](../api/modules/module_benchmark_map.md) との整合）。
