## Part A：Phase I-A 本番（写像 φ：S_asym と 6 軸アノテ）

### A.1 検証命題（再掲）

アテンションの非対称成分 **S_asym = A − A^T** が、**方向付き 6 軸関係スコア**（trust・authority・proximity・intent・affect・history）と**情報量的に**対応するかを検証する。

### A.2 データ設計

| 手順 | 内容 |
|------|------|
| **A2-1 コーパス選定** | 対話ターン単位で、**話者 A・B**が特定できるテキスト。可能なら既存の**関係性ラベル付き**データセットを優先（なければ自前収集＋独自アノテ）。 |
| **A2-2 単位定義** | 分析単位を **(対話 ID, ターン t, 有序ペア (A→B))** に固定。1 ターンに複数ペアがある場合は仕様を一文で定義（例：最終発話者→相手のみ）。 |
| **A2-3 最低件数（v7）** | 文脈タイプ（医療・家族・職場・友人・交渉等）ごと **500 例以上**を目安とする。総数は **最低でも数千ターン**（相関の安定化・部分集合解析用）。 |
| **A2-4 メタデータ** | 各ターンに **context_type**（カテゴリ）、**言語**、必要なら **話者ロール**（上司/部下等）を付与。 |

### A.3 アノテーション設計（6 軸・方向付き）

| 手順 | 内容 |
|------|------|
| **A3-1 ガイドライン** | 各軸の**方向**を明示（例：trust は「A が B をどれだけ信頼するか」）。v7 理論の「非対称／近似対称」の区分に沿い、**authority・intent** は方向性の説明を厚くする。 |
| **A3-2 スケール** | 連続値（例：−3〜+3 または 0〜1）を推奨。閾値・正規化は**事前登録**。 |
| **A3-3 品質** | 複数アノテータ＋**ピルオーバー**（同一サブセットで校正）。合意の取り方（平均／中央値／裁定）を事前に決める。 |
| **A3-4 出力形式** | 機械可読（CSV/JSONL）。列例：`dialogue_id`, `turn_idx`, `speaker_src`, `speaker_tgt`, `trust_ab`, …, `history_ab`（**A→B** と **B→A** の両方を別列で）。 |

### A.4 計算パイプライン（実装タスク）

| 手順 | 内容 |
|------|------|
| **A4-1 トークン整列** | ターン本文をトークナイザで符号化し、**話者スパン**（文字オフセットまたはトークン範囲）と **アテンション行列のインデックス**を対応付ける。 |
| **A4-2 モデル前向き** | 固定モデル（例：GPT-2 / 事前登録した SLM）で `output_attentions=True`。**層**は浅層・中間・深層の**複数**を保存（v7）。 |
| **A4-3 S_asym・S_sym** | 対象層でヘッド平均などで **A** を得て、**S_asym = A − A^T**、**S_sym = A + A^T**（正規化の有無は事前登録）。 |
| **A4-4 ペア特徴** | 話者トークン集合に対応する **クエリ・キーインデックス**から、ペア (A,B) に対応する**部分行列**を集約する方法を固定（例：発話者トークン間の平均 **S_asym**、または [CLS] 相当とトークン間のブロック平均）。 |
| **A4-5 ラベル結合** | 同一 (dialogue_id, turn, pair) で **6 軸スコア**と **S_asym 特徴**（スカラーまたはベクトル）を 1 行にマージ。 |

### A.5 分析

| 手順 | 内容 |
|------|------|
| **A5-1 主解析** | 軸ごとに **Pearson r** と **Spearman ρ**（S_asym 由来の特徴量 vs 各軸スコア）。 |
| **A5-2 補助** | 各軸について **S_asym 特徴 → スコアの線形回帰**（R²）。**S_sym** 側でも同様に計算し、**非対称 vs 対称**の比較表を作る（v7 期待：authority/intent は S_asym 寄り、proximity は S_sym 寄り等）。 |
| **A5-3 多重比較** | 軸数・層数・文脈層別の検定回数を数え、**Bonferroni** または FDR を事前登録。 |
| **A5-4 頑健性** | 文脈タイプ別・層別のサブグループで **r** を再計算。感度分析（別モデル 1 種）を任意で追加。 |

### A.6 判定（v7 設計書）

| 結果 | 次アクション |
|------|----------------|
| **≥3 軸で |r| > 0.3**（補正後） | 「方式 B の情報量的根拠あり」→ Phase IV 深化を検討 |
| **全軸 |r| < 0.1** | 方式 B の見直し・**方式 C**（並列共鳴パス）検討のトリガー |

### A.7 成果物チェックリスト

- [ ] アノテーションガイドライン PDF（版管理）
- [ ] 生アノテ（匿名化済み）＋スキーマ説明
- [ ] 特徴抽出ログ（層・ヘッド集約の仕様）
- [ ] 相関・回帰の表（全体・サブグループ）
- [ ] 事前登録（仮説・閾値・解析コード）のリビジョン

## Part B：Phase III-A 本番（「あわい」測度 A_ij と人間の間合い）

### B.1 検証命題（再掲）

理論上の **A_ij(t)**（v7 定義4.1・Ω 三因子）が、人間の**「間合い」大きさ**評価と**有意に**対応するか（Spearman **ρ**）。

### B.2 シーン設計

| 手順 | 内容 |
|------|------|
| **B2-1 抽出** | 対話コーパスから**関係性の変化が顕著な場面**（転換点）を **200 例以上**抽出。抽出ルールを文書化（キーワード・感情変化・ターン長の閾値・人手スクリーニングの有無）。 |
| **B2-2 対象ペア** | 各場面で評価対象とする**二者**（話者 ID）を固定。 |
| **B2-3 コンテキストウィンドウ** | 評価に含める**前後ターン数**を固定（例：±3 ターン）。**A_ij(t)** を計算する時刻 **t** を定義（例：転換点の直後ターン）。 |

### B.3 人間評価

| 手順 | 内容 |
|------|------|
| **B3-1 アノテータ** | **最低 5 名**（多様性・研修の記録）。 |
| **B3-2 タスク** | 「この二者の間の**間合いの大きさ**」を **7 段階**（定義とアンカー例を提示）。 |
| **B3-3 一致度** | **Krippendorff's α ≥ 0.6** の場面のみ採用（未達は再定義または追加ラベル）。 |
| **B3-4 集約** | 場面ごとに**平均**または**中央値**（事前登録）を人間スコアとする。 |

### B.4 モデル側：A_ij と Ω 三因子の計算

| 手順 | 内容 |
|------|------|
| **B4-1 状態の代理** | 実装レベルで **w_ij(t) ∈ ℝ^d** をどう取得するかを事前に決める。**（a）** 共鳴テンソル推定器（学習済みヘッド）**（b）** 隠れ状態の射影＋合成**（c）** 本番用の Resonanceverse モジュール。いずれも**版とチェックポイント**を記録。 |
| **B4-2 遅延** | **τ_ji** を推定するか、固定遅延（ターン数）で近似するかを事前登録。 |
| **B4-3 A_ij 値** | v7 §4.2–4.3 の **Ω**（または簡略版）を、**時刻 t** で計算。 |
| **B4-4 三因子分離** | ||ẇ_ij||、**1−cos**、**σ(||差||)** をそれぞれログに保存し、後段の偏相関に使う。 |

### B.5 分析

| 手順 | 内容 |
|------|------|
| **B5-1 主解析** | 人間スコア（平均）と **A_ij** の **Spearman ρ**。 |
| **B5-2 偏相関** | 三因子のそれぞれと人間スコアの**偏相関**（他因子を統制）で、どの因子が効いているかを報告。 |
| **B5-3 文脈層** | 感情場面 vs 情報場面などの**サブグループ**（事前定義）で ρ を比較。 |

### B.6 判定（v7 設計書）

| 結果 | 次アクション |
|------|----------------|
| **ρ > 0.4** | 妥当性の最低基準を満たす |
| **ρ < 0.2** | **Ω 関数形の修正**（三因子の重み・定義）を検討 |
| 三因子の偏相関が**いずれも有意でない** | 定義の**構造見直し** |

### B.7 成果物チェックリスト

- [ ] 評価ガイド（7 段階の文言・例）
- [ ] 200+ 場面リスト＋抽出ログ
- [ ] α 計算スクリプトと閾値
- [ ] A_ij・三因子の時系列ログ（再現可能な seed・モデル）
- [ ] ρ・偏相関の表（全体・サブグループ）

## 人手を省略する場合の採用経路（v7 Phase I-A 周辺）

本番（6 軸人手・間合い 7 段階）は**据え置き**とし、次に進むときは次を**正**とする。

| 目的 | 採用するもの | 主張の境界 |
|------|----------------|------------|
| パイプライン検証 | `v7_run_suite.py --demo`、`v7_phase1a_phi_correlation.py --demo` | 合成データの健全性のみ |
| HF ベースライン | `v7_phase1a_phi_correlation.py`（層別 S_asym 統計）、`attn_implementation=eager` | **ラベルとの対応は主張しない** |
| 例示 JSONL | `experiments/data/v7_phase1a_pilot.jsonl` + `v7_phase1a_pilot_jsonl.py` | 数値は**例示**。本番相関の根拠には**しない** |
| **人手なし代理** | `experiments/v7_phase1a_autoproxy.py`：**トークン数（または文字長）**と最終層 `||S_asym||_F` の Pearson | **6 軸の代替ではない**。長さと注意非対称の**粗い共変動**のログ用 |

Phase III-A（人間の間合い）を人手なしで進める場合は、**合成 Ω のみ**（`v7_phase3a_awai_metrics.py`）を継続し、**人間妥当性の主張は行わない**。

## リポジトリ内実証ベースライン（Phase I-A v1）

本番コーパス・人手 6 軸の前に、**固定パイロット JSONL** と **gpt2** でパイプラインを実走させ、成果物 JSON を 1 ファイルにまとめる。

| 項目 | 内容 |
|------|------|
| 事前登録スタブ | [`v7_phase1a_empirical_prereg_v1.json`](v7_phase1a_empirical_prereg_v1.json)（主張の境界・データパス・指標） |
| 一括実行 | [`experiments/v7_empirical_run.py`](../../experiments/v7_empirical_run.py) |
| メタデータ | [`experiments/v7_experiment_meta.py`](../../experiments/v7_experiment_meta.py)（Python / torch / transformers） |

```bash
# CI・オフライン（合成）
python experiments/v7_empirical_run.py --demo --out experiments/logs/v7_empirical/bundle_demo.json

# 実モデル（層別統計・パイロット HF・autoproxy HF）
python experiments/v7_empirical_run.py --cpu --model gpt2 --seed 0 \
  --out experiments/logs/v7_empirical/bundle_hf_gpt2.json
```
