# Phase II-A: 定理「支持」に向けた推奨経路と全体実装の考察

本稿は、[実証実験設計 v7.0 §3.1](../v7/Resonanceverse_v7.0_Experimental_Design.md)・[理論 v7.0 定理 3.3 系](../v7/Resonanceverse_Theory_v7.0.md)・[Phase II-A と理論の橋](v7_phase2a_theory_bridge.md)の文脈で、**定理の実証／数値的支持**をどう位置づけ、リポジトリ全体の実装をどう進めるかの**推奨**と**必要条件**を整理する。

## 1. 用語: 「実証」が指すもの

| 意味 | 内容 |
|------|------|
| **数学的証明** | 理論稿に既に与えられている定理の**論証そのもの**。実装のタスクではない。 |
| **数値的支持（設計書の語彙）** | 理論が予測する **τ\***（または帯）と、**同一モデル族**の数値実験で得る **τ*_exp** が、事前に定めた**乖離基準**（設計書: 20% / 50%）で整合するかを検討すること。 |
| **コーパス上の記述** | MRMP・凍結 LM・注意ベース R(τ) は**別定義域**の統計であり、[橋ドキュメント](v7_phase2a_theory_bridge.md)どおり **定理支持の主証拠に直結しない**（探索的・写像 φ 周辺と位置づける）。 |

混同を避けるため、本文では「**定理支持**」＝数値的・実験的な**整合性検査**、「**実証**」＝文脈によって上記いずれかと読み替える。

## 2. 推奨スタンス（優先順位）

1. **先に整えるべきもの**: 設計書 II-A 型の**同一ダイナミクス**（DDRF または**承認済み簡約**）に対し、**理論 τ\*** を**同じパラメタ記法**で**数値として与える**手順（閉形式・数値根・付録）を固定する。
2. **次に整えるべきもの**: そのモデル上で **V_K**（必要なら Krasovskii **第 2 項**を含む）と **dV/dt の符号判定**を、**離散実装と対応表付き**でログ化する（いまの `½||W||²` スタブは**中間段**）。
3. **後回しにしないが主役にしないもの**: MRMP 実データ・R(τ)・`tau_star_corpus_proxy` は**別図・別節**で、主解析・補助解析として維持する（[事前登録](v7_phase2a_prereg_v1.json)の注記どおり）。
4. **論文表の位置づけ**: [`v7_phase2a_paper_tau_comparison.py`](../../experiments/v7_phase2a_paper_tau_comparison.py) は**合成指標と理論注入の並べ方**のレール。**理論値が未注入なら乖離％は意味を持たない**。

**推奨の一文**: 定理支持を本気で取りに行くなら、**コーパスだけを厚くする前に**、**理論 τ\* の数値化**と **V_K 整合の数値実験**にリソースを寄せる。

## 3. 定理支持に向けた必要条件（チェックリスト）

- **理論参照の固定**: 定理の**条件 1–4**、**μ, N, η（学習率）** 等と **τ\*** の関係が読者に追えること。
- **スケールの同一化**: 連続時間の遅延と、実装の**離散ラグ τ** の対応を**一文の対応表**で固定すること。
- **V_K の実装対応**: 理論の \(V_K\) 項とコードのエネルギー／汎関数が**項ごと**に対応すること（W\*、遅延和、重み \(\tilde\gamma\) 等）。
- **τ*_exp の操作定義**: 「**符号が正に転じた最小 τ**」を**離散系列**でどう判定するか（閾値、burn、ノイズ）を**事前**に固定すること。
- **乖離基準の適用対象**: 20% / 50% を、**どの推定 τ**（単一指標か、感度か）に当てるかを**事前登録**に書くこと。
- **再現性**: コミット・シード・JSON スキーマ・[マニフェスト](EXPERIMENT_ROADMAP_v7.md)に沿ったログ。

## 4. 全体実装を進めるのに必要な事項（横断的考察）

### 4.1 モデル階層の明示

リポジトリには複数階層が共存する。**同じ JSON に載せても同一視しない**ため、スキーマまたは `figure_role` 相当で階層を固定する。

| 階層 | 例 | 主なスクリプト・成果物 |
|------|----|------------------------|
| 合成遅延テンソル（玩具） | `delay_sweep`、Lyapunov スタブ | `v7_phase2a_delay_sweep.py`、`v7_phase2a_tau_exp_lyapunov_stub.py` |
| 理論整合（目標） | DDRF 離散化・V_K 本実装 | **未整備（次段）** — 新規モジュールまたは設計書承認の簡約に依存 |
| コーパス実証 | MRMP、R(τ) | `v7_phase2a_empirical_run.py`、事前登録 |

### 4.2 コードパス

- **橋の 1 JSON**: [`v7_phase2a_theory_bridge_synth.py`](../../experiments/v7_phase2a_theory_bridge_synth.py) — 振動＋α 感度＋メタ。
- **論文比較表**: [`v7_phase2a_paper_tau_comparison.py`](../../experiments/v7_phase2a_paper_tau_comparison.py) — 合成複数指標＋理論注入＋MRMP プレースホルダ。
- **CI**: `v7_run_suite --with-theory-bridge`、Lyapunov スタブ、比較表デモで**退行防止**。**定理支持本体**はパラメタ・コスト次第で**別ジョブ**も検討。

### 4.3 文書・事前登録

- **事前登録の更新**: `simulation_script_reference` に将来の **DDRF／V_K 本番** パスを追加する際は **revision** を上げ、理論稿の節番号と同期する。
- **図表**: [v7_phase2a_paper_figures.md](v7_phase2a_paper_figures.md) で**合成表**と**コーパス図**をキャプション上分離する。

### 4.4 リスクと打ち手

## 5. 既存成果物との関係（読み替え表）

- **振動代理** (`tau_exp_proxy_oscillation_jump`): 感度・形状の**素材**であって τ*_exp ではない（各スクリプトの `design_bridge_ja`）。
- **Lyapunov スタブ** (`tau_exp_numeric_stub_*`): **W\*=0・離散 ΔV** の**操作化**；完全な V_K 手続きの**代替ではない**。
- **MRMP R(τ)**: **コーパス代理**；定理支持の**主柱にしない**。

## 6. 次アクション（実装観点の優先度案）

1. **理論 τ\*** の**数値**と**導出ノート**（リポジトリ内外）の固定。
2. **V_K（＋遅延項）** と **離散 τ 掃引** の **1 本の再現スクリプト**（新規）— 既存 `delay_sweep` からの**移行計画**を本書の§4.1に追記していく。
3. **比較表**への**本番行**の差し替え（理論注入を本番値に、スタブ行を本実装行に）。
4. MRMP は**変更せず**、キャプションと本文の**役割分担**のみ強化。
